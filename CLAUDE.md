This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is **"Is Parallel Programming Hard, And, If So, What Can You Do About It?"** — a large open-source book on parallel programming by Paul E. McKenney. The text is CC-BY-SA-3.0; code samples are GPLv2. The book covers concurrency from basic locking through advanced techniques like RCU (Read-Copy Update), with extensive code examples primarily targeting the Linux kernel ecosystem.

## Build Commands

### Building the PDF

```bash
make            # Build default perfbook.pdf (2-column, letter paper)
make 1c         # 1-column layout
make hb         # Hard-bound book layout
make a4         # 2-column, A4 paper
make eb         # 1-column e-book layout
make nq         # 2-column without inline Quick Quizzes
make sf         # 2-column with sans-serif font
make help       # Show official targets
make help-full  # Show all targets (including ix, df, etc.)
```

### Building Code Samples

```bash
make -C CodeSamples/ pthreads  # Generate platform headers (run first)
make -C CodeSamples/           # Build all code samples
make -C CodeSamples/ check     # Run code sample tests
```

### Checks and Validation

```bash
make precheck      # Verify sed/date/grep compatibility
make punctcheck    # Run punctuation and cleveref checks on LaTeX sources
make figs          # Build all figures (EPS→PDF, SVG→PDF, DOT→EPS)
make clean         # Remove generated files
make distclean     # Clean + remove PDF conversions
```

### Environment Variables

- `PERFBOOK_DEFAULT=1c` — change default `make` target
- `PERFBOOK_PAPER=A4` — set paper size (A4, HB, or letter)
- `LATEX_OPT="-synctex=1"` — enable SyncTeX for editor integration

## Architecture

### LaTeX Document Structure

The main document is `perfbook-lt.tex` which uses custom class `pfbook.cls`. Chapters live in their own directories (e.g., `defer/defer.tex`, `locking/locking.tex`). The build generates variant `.tex` files (e.g., `perfbook.tex`, `perfbook-1c.tex`) from `perfbook-lt.tex` via sed transformations controlled by the Makefile.

Key custom packages:
- `qqz.sty` — Quick Quiz system (inline quizzes with answers collected at end)
- `origpub.sty` — Original publication attribution tracking
- `pfbook.cls` — Custom book class with three separate indexes (General Terms, People, APIs)

### Code Snippet Extraction

Code samples in `CodeSamples/` contain `\begin{snippet}` / `\end{snippet}` markers. The utility `utilities/fcvextract.pl` extracts marked regions into `.fcv` files that are `\input` into the LaTeX source. Dependencies are tracked in `CodeSamples/snippets.d` (generated by `utilities/gen_snippet_d.sh`).

### CodeSamples Structure

- `api-pthreads/` — POSIX threading API abstraction layer (the common API used by all examples)
- `arch-{x86,arm,arm64,ppc64}/` — Architecture-specific atomics and barriers
- `defer/` — RCU variants, hazard pointers, seqlocks, reference counting
- `count/` — Concurrent counter implementations (has most tests)
- `datastruct/` — Hash tables (including resizable), skiplists
- `formal/` — Promela models, herd7 litmus tests for formal verification
- `SMPdesign/` — Parallel maze solver and other design examples
- `lib/` — Shared library code used across examples
- `depends.mk` — Architecture/OS detection; `recipes.mk` — common build recipes

### Build Pipeline

The PDF build runs multiple `pdflatex` passes (via `utilities/runlatex.sh`) interleaved with `bibtex`, `makeindex`, and `makeglossaries` until references stabilize. Figure conversion (SVG→PDF, FIG→EPS→PDF, DOT→EPS→PDF) is handled by Makefile rules in `epstopdf-rule.mk`. The default `all` target builds the PDF then runs `punctcheck-auto`.

### Utility Scripts (in `utilities/`)

- `punctcheck.pl` / `cleverefcheck.pl` — LaTeX source validation
- `fcvextract.pl` — Code snippet extraction from source files
- `gen_snippet_d.sh` — Generates snippet dependency file
- `runlatex.sh` / `runfirstlatex.sh` — LaTeX build drivers with convergence detection
- `extractqqz.sh` — Quick Quiz extraction and division
- `autodate.sh` — Automatic date stamping

## Conventions

### Commit Messages

Follow Linux kernel style: `<component>: <description>`, e.g.:
- `CodeSamples/defer/route_cacm_c11: Make nthreads be _Atomic`
- `defer: Fix grammar issues across Chapter 9 text`

### Contributions

Patches require a `Signed-off-by:` line (Developer Certificate of Origin). Discussion and review goes to `perfbook@vger.kernel.org`.

### LaTeX Style

- UK-style punctuation with quotes (comma/period outside quotes unless part of the quoted material)
- American English spelling, Oxford comma
- Use `\cref`/`\Cref` (cleveref) for cross-references
- Three index categories: general terms, people (with `\IX{}`), and APIs with markers like `(c)` for C standard, `(k)` for Linux kernel
- Templates for figures and code listings are in `.Templates/`
