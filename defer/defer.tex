\Epigraph{好事总会降临到有耐心等待的人身上。}{Violet Fane}

推迟工作的策略可以追溯到有记载的历史之前。
它偶尔被嘲笑为拖延，甚至是纯粹的懒惰。
然而，在过去几十年中，工作者们已经认识到这种策略在简化和优化
并行算法方面的价值~\cite{Kung80,
HMassalinPhD,ValerieAurora2008Synthesis}。
信不信由你，并行编程中的``懒惰''往往能在性能和可扩展性方面
超越勤劳！
这些性能和可扩展性优势源于这样一个事实：推迟工作可以使同步原语弱化，
从而降低同步开销。

那些愿意并且能够阅读和理解本章的人将揭开许多秘密，包括：

\begin{enumerate}
\item	等待并发代码粗心开发者的 reference counting 陷阱。
\label{sec:defer:Mysteries reference-counting trap}
\item	一种并发引用计数器，它不仅避免了这个陷阱，
	而且还避免了昂贵的原子读-修改-写访问，
	此外还避免了对所遍历数据结构的任何类型的写操作。
\label{sec:defer:Mysteries hazard pointers}
\item	被低估的受限形式的软件事务内存（transactional memory），
	在 Linux 内核中被大量使用。
\label{sec:defer:Mysteries sequence locking}
\item	一种同步原语，允许对并发更新的链式数据结构进行遍历，
	使用的机器指令序列与遍历该数据结构的顺序实现完全相同。
\label{sec:defer:Mysteries RCU}
\item	一种同步原语，其使用场景在概念上远比原语本身复杂。
\label{sec:defer:Mysteries RCU Use Cases}
\item	如何在各种延迟处理原语之间进行选择。
\label{sec:defer:Mysteries Which to choose}
\end{enumerate}

工作延迟的一般方法包括
reference counting（\cref{sec:defer:Reference Counting}）、
hazard pointers（\cref{sec:defer:Hazard Pointers}）、
sequence locking（\cref{sec:defer:Sequence Locks}）、
和 RCU（\cref{sec:defer:Read-Copy Update (RCU)}）。
最后，\cref{sec:defer:Which to Choose?}
描述如何在本章涵盖的工作延迟方案中进行选择，
\cref{sec:defer:What About Updates?}
讨论更新操作。
但首先，\cref{sec:defer:Running Example}将介绍一个示例算法，
用于比较和对比这些方法。

\section{运行示例}
\label{sec:defer:Running Example}
%
\epigraph{一盎司的实践胜过一吨的抽象。}
	 {Booker T.~Washington}

本章将使用一个简化的数据包路由算法来展示这些方法的价值，
并允许对它们进行比较。
路由算法在操作系统内核中用于将每个发出的 TCP/IP 数据包
传递到适当的网络接口。
这个特定算法是 BSD UNIX~\cite{VanJacobson88}中使用的经典 1980 年代
数据包列车优化算法的简化版本，
由一个简单的链表组成。\footnote{
	换句话说，这不是 OpenBSD、NetBSD，甚至也不是
	FreeBSD，而是 Pre-BSD\@。}
现代路由算法使用更复杂的数据结构，然而简单的算法有助于在
直观的环境中突出并行性特有的问题。

我们进一步简化该算法，将搜索键从由源和目的 IP 地址及端口
组成的四元组一直缩减为一个简单的整数。
查找和返回的值也将是一个简单的整数，
因此数据结构如\cref{fig:defer:Pre-BSD Packet Routing List}所示，
它将地址~42 定向到接口~1，地址~56 定向到接口~3，
地址~17 定向到接口~7。
这个列表通常被频繁搜索且很少更新。
在\cref{chp:Hardware and its Habits}中
我们了解到，规避不便的物理定律（如光速有限和物质的原子本性）
的最佳方法是对数据进行分区或依赖读多共享。
本章将读多共享技术应用于 Pre-BSD 数据包路由。

\Cref{lst:defer:Sequential Pre-BSD Routing Table}（\path{route_seq.c}）
展示了对应于\cref{fig:defer:Pre-BSD Packet Routing List}的
简单单线程实现。
\begin{fcvref}[ln:defer:route_seq:lookup_add_del:entry]
\Clnrefrange{b}{e}定义了 \co{route_entry} 结构，
\clnref{header}定义了 \co{route_list} 头节点。
\end{fcvref}
\begin{fcvref}[ln:defer:route_seq:lookup_add_del:lookup]
\Clnrefrange{b}{e}定义了 \co{route_lookup()}，它顺序搜索
\co{route_list}，返回对应的 \co{->iface}，
如果没有这样的路由条目则返回 \co{ULONG_MAX}。
\end{fcvref}
\begin{fcvref}[ln:defer:route_seq:lookup_add_del:add]
\Clnrefrange{b}{e}定义了 \co{route_add()}，它分配一个
\co{route_entry} 结构，对其进行初始化并添加到列表中，
在内存分配失败时返回 \co{-ENOMEM}。
\end{fcvref}
\begin{fcvref}[ln:defer:route_seq:lookup_add_del:del]
最后，\clnrefrange{b}{e}定义了 \co{route_del()}，如果指定的
\co{route_entry} 结构存在则将其移除并释放，
否则返回 \co{-ENOENT}。
\end{fcvref}

这个单线程实现作为本章各种并发实现的原型，
也作为理想可扩展性和性能的估计基准。
